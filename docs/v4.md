# Architecture Overview

## Hardware Platform

Computing Unit: NVIDIA Jetson Orin Nano running:

Jetpack 6.X

Ubuntu 22.

ROS2 Humble

Vision Sensor: Intel RealSense D456 depth camera

Output Device: Haptic X40 TactSuit vest (BLE-connected)

## Software Framework

The system is built on ROS2 Humble with 2 main nodes communicating via ROS topics.

# Complete Data Flow: Sensors вҶ’ Haptic Vest

## 1. Sensor Input (RealSense D456 Camera)

The RealSense camera node publishes:

RGB stream (/icansii_v4/camera/color/image_raw) - 848Г—480 @ 30fps

Depth stream (/icansii_v4/camera/depth/image_rect_raw) - 16-bit depth data

Camera intrinsics (/icansii_v4/camera/color/camera_info) - focal lengths,

principal points

IMU data (/icansii_v4/camera/imu/data) - accelerometer & gyroscope

Aligned depth to color for synchronized depth-RGB registration

The camera applies multiple filters (spatial, temporal, hole-fi lling, decimation) to improve

depth quality.

## 2. Object Detection & 3D Reconstruction (yolo_detect_node)


This is the core processing node (yolo_detect.py):

Processing Pipeline:

A. Object Detection

Subscribes to RGB images

Runs YOLO v11-seg (segmentation) using TensorRT engine on GPU

Detects specific classes (currently: вҖңpersonвҖқ)

Extracts bounding boxes and segmentation masks with tracking IDs

B. Depth Extraction & Filtering

For each detected object, computes the center of the mask/bounding box

Samples depth value at that 2D pixel coordinate

Critical: Correctly handles 16-bit depth data (not 8-bit) to get accurate distances

Applies 1D Kalman filter to smooth noisy depth measurements (filters position +

velocity)

C. 3D Point Reconstruction

Uses camera intrinsics (fx, fy, ppx, ppy) to deproject 2D pixel + depth вҶ’ 3D point

Formula: rs2_deproject_pixel_to_point()

Output: 3D position in camera frame (millimeters)

D. Position Stabilization

Applies FIFO buffer (stores last 7 positions)

Uses median filter on position history to reduce jitter

Creates stable 3D coordinates for each object

E. Tracking & Velocity Estimation

Each object gets a MarkerTracker instance with unique ID

Tracks object across frames using YOLOвҖҷs tracking IDs

Computes velocity by comparing positions over time

Maintains separate position and velocity history per object

F. Out-of-FOV Prediction

When object leaves camera view, switches to predictive mode

Uses last known velocity + position to extrapolate trajectory


Predictions valid for up to 3 seconds after loss

Prevents markers from resetting to origin (0,0,0)

Published Topics:

/icansii_v4/yolo_detect_node/markers вҶ’ MarkerArray with detected objects

(green spheres)

/icansii_v4/yolo_detect_node/predicted_markers вҶ’ Predicted positions (red

spheres)

/icansii_v4/camera/fov вҶ’ Camera field-of-view visualization

## 3. Haptic Feedback Generation (vest_ble_node)

This node (vest_ble_node.py) converts 3D detections into tactile vibrations:

Processing Logic:

A. BLE Connection

Establishes async BLE connection using Bleak library

Connects to vest at BLE address FA:28:91:E3:0F:3D

Controls two motor groups via UUID characteristics:

Left motors: 6e400014-b5a3-f393-e0a9-e50e24dcca9e

Right motors: 6e400011-b5a3-f393-e0a9-e50e24dcca9e

B. Marker Selection

Subscribes to /icansii_v4/yolo_detect_node/markers

Filters markers by class whitelist (if configured)

Selects closest valid marker based on 3D Euclidean distance

C. Distance-to-Intensity Mapping

Threshold: 1.25 meters

If object > 1.25m вҶ’ vest OFF

If object вүӨ 1.25m вҶ’ compute intensity:

```
t = (threshold - distance) / threshold # 0.0 to 1.
intensity = min_int + (max_int - min_int) * t # 0-
```

Closer objects вҶ’ stronger vibration

D. Directional Mapping

Uses markerвҖҷs Y coordinate (horizontal position in camera frame):

Y > +0.12m вҶ’ Left side (left motor full, right motor 25%)

Y < -0.12m вҶ’ Right side (right motor full, left motor 25%)

-0.12m вүӨ Y вүӨ +0.12m вҶ’ Center (both motors equal)

Deadzone prevents jitter around center

E. Rate Limiting & Safety

Updates vest at configurable rate (default: 5 Hz)

Watchdog mechanism: If no detection for 0.5s вҶ’ automatically turn vest OFF

Prevents stale vibrations from persisting

# Key Technical Features

1. 16-bit Depth Handling: Correct processing of 16-bit depth images (not 8-bit) for

accurate distance measurements

2. Multi-stage Filtering: Kalman (depth) вҶ’ FIFO buffer вҶ’ Median filter for robust

3D positions

3. Per-object Tracking: Individual velocity/position histories for each detected object

with persistent IDs

4. Predictive Tracking: Continue estimating positions for 3 seconds after objects

leave FOV

5. Haptic Mapping: Intuitive distanceвҶ’intensity + directionвҶ’motor control

6. Safety Mechanisms: Watchdog timer to prevent stuck vibrations

This creates a complete assistive device for visually impaired users, translating visual

obstacles into spatial haptic feedback in real-time on the Jetson Orin Nano platform.
